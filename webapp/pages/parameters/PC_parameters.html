Glossary

Polygon Polygon generated from two parameters: VertexCount and PatternNoise.
VertexCount amount of vertices of the procedural polygon. 3 = triangle, 4 = 4-sided polygon, etc…
PatternNoiseRadius 0 = regular polygon (equilateral triangle, square, pentagon…), 1 = irregular polygon (each vertex will be moved from the regular position).
Stimulus polygons polygons that need to be compared
Distractor polygon polygon that hinders the comparison of stimulus- polygons

Parameters

Stimulus parameters (9 parameters)

StimulusVertexCount

Number of vertices in each polygon-stimulus; both polygon-stimuli have the same PatternVertexCount.

Type: integer Default: 5 Range: 3 - 10

PolygonRegularity

Parameter controls how regular/irregular the generated polygons are (this applies both to stimulus-polygons and distractor-polygons). This is a generic parameter that applies to all types of polygons.

Type: enum Default: Irregular

RotationMagnitude

Magnitude of the rotation applied to the right polygon-stimulus, as a way to make the task harder (mental rotation).

Type: float Range: 0 - 360

HasDiscrepancy

Whether there is a discrepancy between the two polygon-stimuli.

Type: boolean Default: Bernoulli(p=0.5)

DiscrepancyType

In case HasDiscrepancy = True, this variable determines whether the two polygon-stimuli differ on only one vertex or on all of the vertices.

Type: enum Default: SingleVertex

DiscrepancyMagnitude

Describes how much a vertex of a polygon-stimulus is displaced relative to its original position. If all the vertices are changed (i.e., DiscrepancyType = All) that same amount will be applied to all StimulusVertexCount vertices (so the “total difference” between two polygon-stimuli would be ~StimulusVertexCount*DiscrepancyMagnitude). The discrepancy is expressed as a fraction of the maximum displacement allowed by the implementation: a value of 0 means no displacement, 1 means the largest possible displacement (hard coded parameter). Here we don’t want to impose a lower bound.

Type: float Default: 0.4 Range: 0.05 - 1
Note: It may be the case that a value of 0.3 is already very simple: the 0-1 range of DiscrepancyMagnitude does not map to difficulty linearly. We might want to describe discrepancy in the same units as the stimulus-size / stimulus-coordinates. This can be done offline and doesn’t need to be part of the specs.

DistractorPosition

Possible presentation modes/positions for the distractor-polygon.

Type: enum Default: Left

DistractorVertexCount

The number of vertices the distractor polygon has.

Type: integer Default: 4 Range: 3 - 10

ColoredDistractor

Whether the polygon-distractor has a different color than the polygon-stimuli.

Type: boolean Default: "true"

User input parameters (1 parameters)

MaxResponseTime

Time the user has to give an answer, starting from the moment the user can click.

Type: float Default: 20 Range: 1 - 30

Feedback parameters (3 parameters)

ShowSlidingPolygonFeedback

Whether we create a copy of the left stimulus and make it slide on top of the right stimulus so the user can easily compare them visually.

Type: enum Default: OnError

ShowAccuracyOnButtonFeedback

Whether the button highlight shows if the user response is correct/incorrect or instead does not reveal response accuracy. If set to True, the highlight is green for correct responses and red for incorrect responses.

Type: boolean Default: "false"

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always
