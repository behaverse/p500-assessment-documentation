Parameters

Stimulus parameters (10 parameters)

StimulusType

The type of stimulus to display.

Type: enum Default: UpperCaseLetters
Note: Oftentimes people don’t use all possible letters but only a specific subset of letters. These subsets will need to be defined somewhere (probably by the sequence generator itself).

SequenceLength

The number of stimuli to be presented to the user.

Type: integer Default: 60 Range: 1 - Inf

ItemSequence

The actual sequence of symbols to show the user (of size SequenceLength). Only two features of the stimulus are controlled: the identity of the symbol and its foreground color (which is used to signal “distractor”). The values in ItemSequence are indices that refer to a stimulus set defined by StimulusType (i.e., the values [3, 1] could refer to the digits 3 and 1 when StimulusType = Digits and to D and B when StimulusType = UpperCaseLetters). To indicate whether the symbol is a distractor or not, the method used here is to assign a negative value to distractors.

Type: sequence of integers Range: -PoolSize-1 - PoorSize
Note: PoolSize value depends on StimulusType (e.g.,10 for digits, 26 for the alphabet). A positive value represents a stimulus; a value of 0 is “0” in digits and “A” in letters A negative value represents a distractor, which value is equal to (- ItemSequence - 1) (i.e., in digits, -5 is “4”, -1 is “0”) For SART, distractors do not exist, therefore range is [0, PoolSize]

StimulusColor

The color of the main stimuli.

Type: color Default: #F5DFC0
Note: The default color is close to white.

DistractorColor

The color of the distractors.

Type: color Default: #618DA6
Note: The default color is blueish.

StimulusDisplayDuration

The duration in seconds during which each stimulus is visible.

Type: float Default: 0.1 Range: 0.01 - Inf

InterstimulusInterval

The duration in seconds between each stimulus, Starting from the end of StimulusDisplayDuration.

Type: float Default: 0.9 Range: 0 - Inf

NoiseMaskDisplayDuration

The duration of displaying NoiseMask in seconds, starting right after StimulusDisplayDuration ended.

Type: float Default: 0.3 Range: 0 - Inf
Note: The mask is a static noise texture (gray levels).

ResponsePatterns

What patterns should be searched for.

Type: array of String Default: ["A"]
Note: Pattern operators (following classic regexp syntax): : a single character ! : Any character except the one given . : One character ? : Zero or one character (except the next character after the question mark) * : Any number of any characters (except the next character after the star) Examples: X: Press when X appears !X: Press when anything except X appears (e.g. “A”, “B”) AX: Press when X appears right after an A (e.g. “AX”) A!BX: Press when X after A followed by anything but a B (e.g. “AFX”) A..X: Press when X appears after A and two letters (e.g. “ACFX”, “AJAX”) A??X: Press when X appears after A and from 0 to 2 letters (e.g. “AX”, “AFX”, “AJAX”) A*X: Press when X appears after A and any number of letters (e.g. “AX”, “AFX”, “AJAIX”) Note: Patterns cannot start nor end with “?”, “.” or “*” Patterns can be written with upper-case letters or digits (e.g. “A!X” or “1?5”) For SART, Patterns consist of only one symbol or its exception: “X” or “!X”

HidePatternTrialDelay

After how many trials we hide the patterns on screen.

Type: integer Default: 9999 Range: 1 - Inf

User input parameters (1 parameters)

UseNonMatchButton

Whether to display a “Non-Match” button along with the “Match” button. The idea here is to disambiguate user input: when there are no non-match buttons, an absence of a response could mean the user reports a non-match or she did not pay attention to the stimulus.

Type: boolean Default: "false"
Note: If ResponsePatternCount > 1, the number of buttons are multiplied, therefore having a Non-Match button is discouraged for more than one stream. In SART, is always false.

Feedback parameters (4 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of an episode (i.e., sequence of symbols).

Type: enum Default: Always

ShowAccuracyOnButtonFeedback

What type of feedback we show on the button itself once it is pressed.

Type: enum Default: Never

ShowCorrectTrialResponseFeedback

Whether we should show the correct response feedback at the end of a trial.

Type: enum Default: Never

ShowBlockStats

In the current build 1.0.10 at the end of the trial displays a percentage correct. This is not described here; that feedback should not be colored red/green.

Type: boolean Default: "false"
