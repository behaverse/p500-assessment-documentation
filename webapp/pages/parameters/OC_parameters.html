Glossary

SIO a simple interactive object, like a button but which can be in an on/off state and display a colored symbol on a colored background (like the cards in the card sorting test).
Trial corresponds to a configuration of SIOs and an order of SIOs. How a trial ends depends on the particular case: it might end when the user clicked on all SIOs in the correct order, when the user made an error or time ran out (see below).
SIOCount number of SIOs to be displayed during a trial on the screen.
SIOStreamCount number of different categories of SIOs to be used in a trial. For SIOStreamCount = 2, one category could be digits and the other letters. SIOStreamCount is not a parameter but instead is defined as the length of StreamFeatures.
Feature category of ordered symbols to be displayed on a sequence of SIO (e.g. digits, letters).

Parameters

Stimulus parameters (23 parameters)

SioCount

The number of SIOs to be displayed in total. This number should be a multiple of the number of streams.

Type: integer Default: 24 Range: 1 - 49

Streams

The different sequences of symbols/SIOs to track. For example, in part A of the Trail Making Task (TMT) there is only one stream, and users have to click on the SIOs numbered from 1 to 25. In part B, there are 2 streams and users have to alternate between clicking on digits and letters in increasing order.

FeatureType

The type of the feature to use.

Type: enum Default: Digits

SioBackgroundColor

The background color of the Sios (colors are defined in the Colors section above).

Type: color Default: “light-gray”: rgb(143, 156, 155)

SioTextColor

The text color on the Sios.

Type: color Default: “dark-gray”: rgb(56, 62, 71)
Note: Only for OC & ML.

AscendingOrder

Whether the enumeration starts from 1 (or equivalent) and up, or the other way.

Type: bolean Default: "true"
Note: Only for OC & ML.

TrackStream

Whether this stream should be tracked by the player. Otherwise, it should be ignored.

Type: boolean Default: "true"
Note: At least one stream should be tracked. Otherwise, by default, the first stream is tracked.

OrderingRule

Which rule to use to define the order in which the SIO should be clicked. The indices and order of the SIOs within a Stream are defined by StreamFeatures (startValue and increasingOrder).

Type: enum Default: Sequential
Note: Only for OC & ML.

IgnoreErrorsForFirstSio

Whether all incorrect answers are ignored until the first correct SIO is clicked by the user.

Type: boolean Default: "false"

FirstClickAction

Determines if a visual change should happen to all symbols after the first SIO (the first correct if IgnoreErrorsForFirstSio=true) has been clicked.

Type: enum
Note: Only for OC & ML. When FirstClickAction == ShowAllSymbols, the first two elements in the sequence will be visible from the start of the trial. This is because we want people to see the pattern they are supposed to follow (from the first two of the sequence) and we don’t want them to prepare as long as they want to process all the sequence (hence it is hidden until they press to “start”). Once the first correct element is clicked, all SIO will be shown.

HideSymbolsDelay

How long the Symbols should be displayed on SIOs after trial onset.

Type: float Default: 3600 Range: 0 - Inf
Note: Only for OC & ML.

SioOrderPresentationDisplayDuration

The duration in seconds of each stimulus being presented sequentially before the user can respond. Useful for SS, but available to OC and ML.

Type: float Default: 0; 1.5 for Spatial Span Range: 0 - Inf
Note: If set to 0, then the SIO presentation is skipped entirely

SioOrderPresentationInterval

The interval between the presentation of two consecutive SIOs. Useful for SS, but available to OC and ML.

Type: float Default: 0; 1 for Spatial Span Range: 0 - Inf
Note: Is ignored if SioPresentationDisplayDuration == 0

ReversedSioOrderPresentation

Whether we show the SIOs in reverse. This parameter is needed for the “backward” version of the Corsi-block/Spatial Span.

Type: boolean Default: "false"
Note: Only for SS (Spatial Span). The logic here is that the stimuli are presented in reverse order, but participants have to report the SIO in the correct order; an equivalent but less elegant strategy would be display one order and reverse the expected responses.

ClickedSioVisualState

What the SIO should look like once it has been clicked on.

Type: enum
Note: when ClickedSioVisualState == Pressed, one can press on each SIO only once. If it’s none you can click multiple times on the same SIO.

HighlightSio

Which SIOs should be highlighted after a short inactivity duration defined by HighlightSioDelay. This highlight is a hint to help players who didn’t understand the rules.

Type: enum

HighlightSioDelay

A delay before highlighting a SIO.

Type: float Range: 0 - Inf

StopTrialCondition

When the trial should be ended.

Type: enum Default: OnError
Note: Note: AfterSIOCountResponses is needed for the Spatial Span task where a sequence of locations are shown and participants have to reproduce that sequence. Note: AfterSIOCountResponses only works in SpatialSpan config right now. Note: When StopTrialCondition = AfterSIOCountResponses, ClickedSioVisualState must be None. Otherwise, odd scenarios will happen while clicking on the Sios , especially in case of clicking a Sio twice. If ClickedSioVisualState != None then the game forces the value to be Null and print a Log Error. Note: In the future, we may need both parameters to work together (e.g. user has to click N buttons, but they still disappear when we click on one). Note: This works ok now. But in the future if we want to have repetition ( clicking the same Sio multiple times according to the pattern itself) then this rule must be applied.

Layout

Determines how the SIOs should be distributed in space.

Type: enum

GridSize

Determines the grid size. If it’s too small to contain all SIOs, then this parameter is ignored and the grid will be the smallest size possible to contain all SIOs.

Type: integer Default: 4 Range: 2 - 6
Note: When using a grid, the grid size will be computed from the SIO count, using this formula if gridSize is too small: GridSize = Ceil(Sqrt(SioCount))

FillGrid

Whether we want to fill the grid with empty Sios . This is useful for the Spatial Span task.

Type: boolean Default: "false"

AllowGridDiagonals

Whether we can go diagonally in the grid when Layout is GridConnection.

Type: boolean Default: "false"

PresetCoordinates

An array of predefined coordinates of the stimuli. These coordinates are used in Layout= Preset. The coordinates should be normalized positions within the scale [0,1] where [0,0] is the bottom left of the board square and [1,1] is the top right of it.

Note: If there are multiple streams, the Sios are given in the order the user needs to click them. For example, if OrderingRule=Alternate, the first Sio will be 1, the second A, the third 2, etc. Coordinates are expressed relative to central board not the screen overall, with lower left corner of the board being (0,0) and upper right corner being (1,1).

User Input Parameters (2 parameters)

EnableInputDelay

The delay after the SIO presentation has ended before the user can press the Sios.

Type: float Range: 0 - Inf

MaxResponseTimeAfterFirstClick

The maximum time users have to complete the trial after the first correct Click; after this duration the trial is ended.

Type: float Default: 20 Range: 0 - Inf

Feedback parameters (4 parameters)

DrawTrails

Whether a trail should be drawn to connect two SIOs when a SIO is clicked and is the next in the sequence.

Type: boolean Default: "false"

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always

ShowAccuracyOnButtonFeedback

Whether we color the outline of the Sio red or green to indicate whether the response was correct or incorrect. If false, the outline will be white.

Type: boolean Default: "true"

ShowCorrectTrialResponseFeedback

Whether we should show the correct response feedback at the end of a trial.

Type: enum Default: OnError
