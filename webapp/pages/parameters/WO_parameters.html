Glossary

Primary Feature the feature (either color or symbol) which is used to categorize the stimulus.
Secondary Feature the feature (different from primary; either color, symbol, position or flankers) which is used to create congruent, incongruent or neutral trials.
Stimulus one or more visual elements displayed on the screen; one main element defined by its color, symbol and position (left, right or center), plus optional flankers elements which are copies of the main element that may differ along the primary feature dimension.
Flanker A copy of the stimulus main element, that may differ along the primary feature dimension; displayed alongside the main element (same line).
Congruent trial trial where the value of the secondary feature for the stimulus maps to the correct button (i.e., both primary and secondary features map to the same button);
Incongruent trial trial where the value of the secondary feature for the stimulus maps to the incorrect button;
Neutral trial trial where the value of the secondary feature for the stimulus does not map to any button;

Parameters

Timing parameters (5 parameters)

FixationPointDisplayDuration

Defines how long the fixation point stays on screen since trial onset.

Type: float Range: 0 - Inf

StimulusOnset

When the stimulus should be shown in seconds relative to trial onset.

Type: float Default: 1 Range: 0 - Inf

StimulusDisplayDuration

Defines how long the stimulus is shown since StimulusOnset.

Type: float Default: 1 Range: 0 - Inf

InterTrialInterval

The duration in seconds between each trial, from the user input to the fixation point onset.

Type: float Default: 0.5 Range: 0 - Inf

TaskPace

Whether the pace of the game is determined by the player or not.

Type: enum Default: PlayerDriven

Task parameters (8 parameters)

PrimaryFeature

Which feature the user should focus on.

Type: enum Default: Symbol

SecondaryFeature

Which feature of the stimulus to manipulate to create the congruent, incongruent or neutral conditions.

Type: enum Default: Color
Note: SecondaryFeature must be different from PrimaryFeature.

Congruency

Whether the stimulus is congruent with the buttons, alongside the secondary feature.

Type: enum Default: Uniform distribution over possible values.

CorrectButton

Which button is the correct one to press.

Type: enum Default: once per trial

ButtonSymbols

The symbols on the buttons, the first being for the left button, the second for the right button. See Behaverse Symbol Names for the full list of symbols.

Type: BehaverseSymbols Default: Circle
Note: Note: the two buttons can display the same symbols (e.g., square). Note: specified in file as ButtonSymbols: [“Circle”, “Circle”]

ButtonColors

The colors for the button, the first being for the left button, the second for the right button.

Type: color Default: White
Note: Note: the two buttons can display the same colors (e.g., white). Note: specified in file as ButtonColors: [“#ffffff”, “#ffffff”]

NeutralSymbol

The symbol used for: - The stimulus if the primary feature is not Symbol, and either the congruency is neutral or the secondary feature is also not Symbol; - The flankers if the primary feature is not Symbol or if congruency is neutral.

Type: BehaverseSymbols Default: Circle

NeutralColor

The color used for: - The stimulus if the primary feature is not Color, and either the congruency is neutral or the secondary feature is also not Color; - The flankers if the primary feature is not Color or if congruency is neutral.

Type: color Default: White
Note: Note: In this task we don’t specify the stimulus directly. Instead we specify the buttons and primary/secondary features and congruency and the stimulus is generated from the sets that are implied by those variables. For example if ButtonSymbols are the left and right arrow, and ButtonColors are black and black, and PrimaryFeature = “Symbol” and SecondaryFeature is “Flanker” and correctButton = “leftButton” and Congruency = “Congruent”, then the stimulus will be a black left arrow surrounded by black left arrows. Note: It is not immediately obvious what stimulus is shown given the task parameters; in order to make this clear, the saved data (stimulus presentation event) should contain an explicit description of the actual stimulus (see StimulusShow below).

User input parameters (3 parameters)

ButtonCount

The number of buttons on screen.

Type: int Default: 2 Range: 1 - 2

ShowButtonContent

Whether the button content is shown.

Type: boolean Default: "true"

MaxResponseTime

The maximum time the user can take to answer. After that, the trial is considered failed.

Type: float Default: 5 Range: 0 - Inf
Note: Note: Participants can respond as soon as the stimulus appears.

Feedback parameters (2 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial

Type: enum Default: Always

ShowCorrectTrialResponseFeedback

Whether we should highlight the correct button at the end of a trial.

Type: enum Default: Never
