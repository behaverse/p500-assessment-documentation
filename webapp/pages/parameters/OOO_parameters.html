Glossary

Set the collection of stimuli presented to the user
Stimulus an element of the Set, which can be distinguished by its features
Features color, number (= amount of components) and shape
Odd One the stimulus of the set that can’t be matched to other stimuli of the set
Rule a set of constraints used to group the stimuli, and exclude the Odd One. The constraints are : share color, share shape, share number.
Group a subset of the stimuli that are grouped according to the rule

Parameters

Constants (2 parameters)

SetSize

The number of stimuli in the set.

Type: integer Default: 9

MaxDifficultyLevel

The number of distinct difficulty levels (see "Difficulty modulation").

Type: integer Default: 13

Generation parameters (5 parameters)

DifficultyLevel

This parameter controls the difficulty by setting the value for GroupCount, GroupFeatureMatch and OddOneExclusiveFeature. Each value of DifficultyLevel maps to different values of these 3 parameters. See Difficulty Modulation.

Type: integer Default: 1 Range: 1 - MaxDifficulty
Note: Difficulty can be modulated by modifying multiple parameters together: - Increasing total item count: more combinations means more information to process. - Increasing GroupCount: more distinct groups, of smaller size. - Having groups with GroupFeatureMatch = false: identical items are easy to pair. - Avoiding exclusive properties on the Odd One: unique items are easy to find (OddOneExclusiveFeature). It's not guaranteed that this manipulation will work as expected.

FeatureOrder

Define the order of the 3 features. The first two features will be used to form groups. The first feature will be the exclusive one if OddOneExclusiveFeature = true.

Type: Array of feature with values Color, Symbol, Number, ordered randomly (Default). Can also be a fixed order (eg: for tutorial, Color can be first to make it easier).

[GroupFeatureMatch]

f true, items of a group are identical. If false, items of a group only share the feature needed to form the group, the other features can differ within the group.

Type: boolean Default: "false"
Note: This parameter is currently not exposed. Its values are defined by DifficultyLevel

[GroupCount]

The number of groups in the set.

Type: integer Range: 1 - 4
Note: This parameter is currently not exposed. Its values are defined by DifficultyLevel. 1: one group of 8 items 2: two groups of 4 items 3: three groups : 3-3-2 items 4: four groups of 2 items.

[OddOneExclusiveFeature]

Define if the Odd One has an exclusive feature value or not. If not, each of its features values is shared by at least one other object in the set.

Type: enum
Note: This parameter is currently not exposed. Its values are defined by DifficultyLevel. If GroupCount == 1, there can be no exclusive feature.

User input parameters (1 parameters)

MaximumIdleTime

The maximum allowed time to answer. After that, the trial is considered failed.

Type: float Default: 30 Range: 1 - Inf

Feedback parameters (4 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial

Type: enum Default: Always

ShowCorrectTrialResponseFeedback

Whether we should show what the correct response would have been at the end of a trial.

Type: enum Default: Never

ShowPairingFeedback

Whether we should show users at the end of the trial how the various stimuli on the screen group together according to the current rule (this is implemented using a movement animation and position cards that go together closely in space; see Feedback presentation).

Type: enum Default: Never

PairingDisplayDuration

Maximum duration of the pairing feedback (see above). The user can skip the feedback.

Type: integer Default: 10 Range: 0.1 - Inf
