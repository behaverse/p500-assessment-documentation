Parameters

Constants (1 parameters)

MaxItemCount

The maximum number of items that users can see on the screen.

Type: integer Default: 20

Stimulus parameters (14 parameters)

ItemCount

The number of items displayed on the screen.

Type: integer Default: 8 Range: 1 - MaxItemCount

TargetCount

The maximum number of targets that the user should track during the game trial.

Type: integer Default: 4 Range: 1 - MaxItemCount

TargetDisplayDuration

The duration in seconds during which the targets are visible to the user (i.e., the user sees the content of the dots; you are told which dots to track). At the end of this duration, targets will become identical to the non-targets.

Type: float Default: 2 Range: 0.01 - Inf

MovementStartDelay

After the initial presentation of targets, the delay after which all items start moving.

Type: float Default: 2 Range: 0 - Inf
Note: targets can still be visible when movement starts, if TargetDisplayDuration > MovementStartDelay.

MovementDuration

For how long (in seconds) the elements move.

Type: float Default: 4 Range: 0 - Inf

MovementSpeed

How fast the elements move in screenHeight/second. Speed could be set to 0. Speed is typically set to a constant value within a trial; there’s no modulation of speed by the objects colliding—this feels rather unnatural but might be “cleaner” from an experimental point of view.

Type: float Default: 4 Range: 0 - Inf

DotRadius

The radius of the dot, in unity units.

Type: float Default: 0.2 Range: 0 - 0.5
Note: the background disk has a radius of 6 uu (unity units); the collider (actual physical boundary, not visible) has a radius of 4.33 uu.

CollisionRadiusMultiplier

The radius of the collision circle, relative to the size of the dot. A radius of 1 means the visual representation of the dot fits its collision radius. A radius of 1.5 means the collision area radius will be 1.5 times the radius of the dot. A radius of 0 means that items do not collide at all (i.e., they move through each other).

Type: float Default: 1 Range: 0 - 3
Note: This parameter does not affect the collision with the fixation point, but only the collision with other items.

DirectionChangeProbability

The probability for each element to change movement direction on any frame (60Hz) (this avoids changes of motion to be synched). 0: the dot will always go straight (unless a collision occurs); 1: there is a change every single frame.

Type: float Range: 0 - 1

DirectionChangeMagnitude

How much (absolute value of angle in degrees) a dot direction can change relative to its current trajectory; the actual direction change angle will be sampled from the range [-1;1]*DirectionChangeMagnitude: 0° : no change 180° : all new directions are possible, past history is erased

Type: float Range: 0 - 180

HasFixationPoint

Whether we need to show a fixation point or not. (This might be handy for having the safe area around the fixation point as it has been mentioned above).

Type: boolean Default: "true"

FixationPointColliderRadius

The radius of the circle collider of the fixation point, to prevent items from floating over it.

Type: float Default: 1 Range: 0 - 2

ItemContainerLayout

The board layout containing the items.

Type: enum Default: Circle

ProbeIsTarget

In YesNo mode, whether the probed item is a target.

Type: boolean Default: Bernoulli(p=0.5)

User input parameters (4 parameters)

ResponseStartDelay

The amount of time (in seconds) to wait between the end of the dot movement and the moment the user is asked to respond.

Type: float Default: 2 Range: 0 - Inf

MaxResponseTime

The duration in seconds during which the user can answer. After this delay, the response is marked as “timed out”.

Type: float Default: 999 Range: 0.01 - Inf

ResponseMethod

Different ways of asking the user about what he saw during the trial.

Type: enum Default: YesNo

QuestionCount

In YesNo mode, the number of questions the user needs to answer within each trial.

Type: integer Default: 1 Range: 1 - MaxItemCount

Feedback parameters (3 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always

ShowResponseFeedback

Whether we show the colored highlight on the clicked button(s) and reveal the probed item(s) when the user clicks a button.

Type: boolean Default: "true"

RevealTargetsOnTrialEnd

Whether we should reveal all the diamonds at the end of a trial.

Type: enum Default: OnError
