Glossary

Card A pattern with a defined number of features (typically Number, Shape and Color). The screen displays multiple cards, they are either the stimulus card or the reference cards see below.
Feature One characteristic that can be used for the sorting (e.g., color).
Stimulus A card that needs to be sorted.
Reference set A set of cards all having unique feature levels (i.e., if one card has color = “red”, none of the other cards can have color = “red” but must pick a color from the remaining levels of color).
Reference card the reference set contains a set of cards which individually are called “reference card”
Rule The feature or conjunction of features that assign each Card to any of the Cards in the Reference. This is sometimes called “principle” or “set” in the literature.
RuleCue A cue that indicates what the current rule is. Can be a text and/or an icon. For now we use text only.
Trial An episode that starts with the presentation of the cards and asks for one response. The number of trials corresponds to the number of cards to be classified.
Rule Switch Trial trial during which the rule has changed relative to the previous trial
Rule Segment A group of consecutive trials with the same rule. A batch start with a Rule Switch Trial, and end just before the next one. This is also sometimes called a “run”.

Parameters

Constants (2 parameters)

MaxFeatureCount

The maximal number of features that a card can have. This parameter is fixed by design, because each feature needs to be recognizable from one to another.

Type: int Default: 4
Note: The number of maxFeatureCount is 4: - Symbol (e.g., 0) - Color (e.g., orange) - Number (e.g., 2) - Background (e.g., diagonal lines)

MaxReferenceCount

The maximal number of reference cards to be shown on screen. This number is constrained by the number of cards that can be displayed on the screen (in terms of size) and will determine the minimal number of levels (i.e., different values) a feature must have. For example, if we display 6 reference cards we need the features (e.g., color) to have 6 different values so each reference card has its own (here color).

Type: int Default: 5

Session parameters (17 parameters)

FeaturePool

A sequence which determines which features can be used to generate the cards (e.g. [ Symbol, Color ] for 2 features). The Features themselves can take on a limited number of values

Type: enum Default: [Symbol, Color, Number]
Note: Features that are not used receive constant values (e.g., if Number is not used, each card will display only 1 symbol).

ReferenceCount

Number of reference cards in the reference set.

Type: int Default: 3 Range: 2 - MaxReferenceCount
Note: In easier versions of the task for kids, it is usual to set Count(FeaturePool) = 2 (e.g., symbol and color) and to only have two levels for each (and hence only 2 cards in the Reference Set.

CardSymbols

The symbols used for the reference cards.

Type: BehaverseSymbols Default: [ "Disk", "Triskelion", "HollowSquare", "Cloud", "Hourglass" ]
Note: If Symbol is not in the feature pool, the first symbol of the list will be used as the default symbol.

CardColors

The colors used for the reference cards.

Type: color Default: [ "#F0B40A", "#554BE1", "#D24B4B", "#14C8C8", "#F0F0F0" ]
Note: If Color is not in the feature pool, the first color of the list will be used as the default color.

CardNumbers

The number of symbols used for the reference cards.

Type: int Default: [ 1, 2, 3, 4, 5 ] Range: 1 - 9
Note: If Number is not in the feature pool, the first number of the list will be used as the default number.

SymbolSize

Defines the size of the symbols, whether a normal size or big size, which can be useful when there is only one symbol in the card.

Type: enum Default: Normal

CardBackgrounds

The background patterns used for the reference cards.

Type: BackgroundPattern Default: [ "Lines", "Circles", "Waves", "Staircase", "Grid" ]
Note: If Background is not in the feature pool, the first background of the list will be used as the default background.

RequiredConsecutiveSuccesses

The number of consecutive successes required before changing the sorting rule.

Type: int Default: 6 Range: 1 - Inf
Note: The rule will change if successes = RequiredConsecutiveSuccesses OR segmentIndex = RuleSegmentLength.

RuleSegmentLength

The number of trials (within the same rule segment) required before changing the sorting rule.

Type: int Default: 999 Range: 1 - Inf
Note: The rule will change if successes = RequiredConsecutiveSuccesses OR segmentIndex = RuleSegmentLength.

InitialRule

Determines how the first rule is picked.

Type: enum Default: FirstChoiceIncorrect
Note: Some studies make it so that the first pick the users make (e.g., they classify based on color) is the correct one; then after N correct trials they switch. An alternative would be to have the first always be incorrect.

ShowCardContent

Whether the card content is shown or not. If not, all buttons will look similar.

Type: boolean Default: "true"

RuleCueOnset

After trial start, delay after which the rule is displayed. The rule cue is displayed after RuleCueOnset seconds, for a duration of RuleCueDisplayDuration seconds.

Type: float Range: 0 - Inf

RuleCueDisplayDuration

Display duration of the rule cue. The rule cue is displayed after RuleCueOnset seconds, for a duration of RuleCueDisplayDuration seconds.

Type: float Range: 0 - Inf

UpperCaseRuleCue

Is the rulecue text uppercase, or not. This is used to add some variety to the cue (avoid automatic response).

Type: boolean Default: "false"

StimulusOnset

After trial start, delay after which the stimulusCard is displayed.

Type: float Range: 0 - Inf

StimulusOffset

When the stimulus is hidden.

Type: enum

MaxIdleTime

The maximum time the user can wait to make his answer.

Type: float Default: 10 Range: 0 - Inf

Generation rules (4 parameters)

StimulusGenerationRule

Defines how the stimulus is generated.

Type: enum Default: Balanced

AvoidExactMatch

Prevents the stimulus card to correspond exactly (every feature) to a reference card.

Type: boolean Default: "true"

EnsureDifferentSuccessiveCards

Ensures successive cards are visually different.

Type: enum Default: OneFeature
Note: According to the test manual (p19), “no two response cards in succession have the same color, form or number.”

ChangeCardOnError

Whether we should draw another card (meaning changing the stimulus card) in case of a failure (i.e., the stimulus card did not match the reference card the user assigned it to). If false, the same card stays on the screen until the user successfully sorts it.

Type: boolean Default: "true"

Feedback parameters (5 parameters)

ShowAccuracyCheckFeedback

Whether we show the valid or wrong “check” feedback at the end of a trial on the screen center.

Type: enum Default: Always

ShowCorrectReferenceOnError

Whether we should show the user which Reference card would have produced a correct response when the user is incorrect.

Type: boolean Default: "false"

ShowCurrentRuleFeedback

At the end of the trial, whether we should inform the user about what the correct sorting rule is (e.g., “color”)

Type: enum Default: Never

HighlightRuleOnSwitch

Whether a change in the sorting rule should be signaled to the user (by applying a tremor effect on the new card to sort).

Type: boolean Default: "false"

ShowAccuracyOnButtonFeedback

Whether a feedback is shown if the user is correct or not when he presses a button. This feedback would indicate correct/incorrect for each press by changing the color of the border, but would NOT reveal any stimulus (i.e., FeatureValue, e.g., Color).

Type: boolean Default: "false"
Note: if ShowButtonFeedback = True, at button click, outline of the buttons is colored red or green for error and correct responses, otherwise button outline is white.
