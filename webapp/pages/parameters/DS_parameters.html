Parameters

Constants (1 parameters)

MaxStimuliOnScreen

The maximum number of stimuli that can be displayed on the screen at once.

Type: integer
Note: This should be at least 12 (the maximum score for players on CBS DigitSpan task).

Stimulus Parameters (9 parameters)

StimulusType

The type of stimulus to display.

Type: enum Default: Digits

SequenceLength

The number of stimuli for the user to remember.

Type: integer Default: 3 Range: 1 - Inf
Note: Normal people remember around 7 but with training/technique may remember up to 80 stimuli, the world record being 465.

ItemSequence

The sequence to show the user.

Type: integer sequence Range: 0 - PoolSize-1
Note: It should be a sequence of SequenceLength; PoolSize value depends on StimulusType, either 10 or 26.

StimulusPackSize

How many stimuli are displayed to the user simultaneously. For example, a pack size of 2 could produce this sequence: “12 / 73 / 19”

Type: integer Default: 1 Range: 1 - 5
Note: 5 is the maximum we will support showing on the screen. We will usually use a size of 1 or 2, going to 5 may already be overkill.

StimulusDisplayDuration

The duration in seconds during which each stimulus is visible.

Type: float Default: 0.5 Range: 0.01 - Inf

InterstimulusInterval

The duration in seconds between each stimulus

Type: float Default: 0.5 Range: 0 - Inf

ColorScheme

The type of colorization of the stimuli.

Type: enum Default: Neutral

ColorMapping

The mapping of a stimulus value to a color.

Type: enum Default: Forward

ColorSaturation

The color saturation, 0 being gray and 1 being vivid colors.

Type: float Default: 1 Range: 0 - 1

User Input parameters (8 parameters)

RetentionDelay

The delay after all stimuli have been shown (i.e., offset of the last stimulus) before the input panel is enabled, in seconds.

Type: float Default: 1 Range: 0 - Inf

MaxIdleTime

The maximum time the user can be idle. After that, the trial is considered failed.

Type: float Default: 30 Range: 1 - Inf
Note: Note that the idle timer resets only when a new input has been made by the user, but not when the user erases a character to write another character.

ColoredButtons

Whether the symbols on buttons are presented with the color they were shown (e.g., if “5” was red, then the “5” key will be red as well) or their default color.

Type: boolean Default: "false"

RevealStimuli

Whether the stimuli are revealed or hidden.

Type: boolean Default: "false"

ShowUserInputSequence

Whether we want to show on screen the current input sequence as it’s typed by the user.

Type: boolean Default: "true"

RecallOrder

The order in which the player should recall the sequence (e.g., 1, 2, 3, 4)

Type: enum Default: Forward

HighlightCorrectButton

Which SIOs should be highlighted after a short inactivity duration defined by HighlightSioDelay. This highlight is a hint to help players who didn’t understand the rules.

Type: enum

HighlightCorrectButtonDelay

A delay before highlighting the next correct button.

Type: float Range: 0 - Inf

Feedback parameters (2 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always

ShowCorrectTrialResponseFeedback

Whether we should show the correct response feedback at the end of a trial.

Type: enum Default: OnError
