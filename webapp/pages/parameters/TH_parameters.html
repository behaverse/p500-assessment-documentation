Glossary

Stimulus A white disk
Finger disk A white disk showing the starting point for the finger
Stop Signal A signal that appears when the target and finger placeholder mostly overlap
Stimulus Placeholders A placeholder the target stimulus can appear on

Parameters

Constants (1 parameters)

MaxPlaceholderCount

The maximal number of placeholders that can fit on the screen.

Type: integer Default: 5 Range: 5 - 5

Input data (1 parameters)

IsStopSignalTask

Indicate if we are doing a task with shields or not (used for skipAdaptiveAlgorithm).

Type: boolean Default: "false"
Note: we will remove this parameter once we either separate the config file into two tasks, or once we can define conditions for adaptive in psych.

Stimulus parameters (5 parameters)

PlaceholderCount

The number of possible locations the target might appear at.

Type: integer Default: 3 Range: 1 - MaxPlaceholderCount

ShowPlaceholders

Whether we want to show the placeholders.

Type: boolean Default: "true"

StimulusType

How the stimulus is presented to the player.

Type: enum Default: VisualAndAuditory

TargetPlaceholderIndex

The index of the placeholder in which the target will appear (index starting from 1 to N).

Type: integer Default: Uniform random sampling Range: 1 - PlaceholderCount

HasShield

Whether the current target will have a “stop” shield when the user approaches it, meaning he has to abort the action and not drag the circle onto it.

Type: boolean Default: "false"
Note: The shield does not have a collider and is only a visual cue.

Timing parameters (3 parameters)

StimulusDisplayDelay

When the stimulus appears after the user pressed his finger on the disk (“foreperiod”).

Type: float Default: 1 Range: 0 - Inf

ShieldActivationDelay

If the target has a shield, the delay after the stimulus has been displayed after which the shield will be shown.

Type: float Default: 0.25 Range: 0 - Inf
Note: The shield may activate earlier than this, at the moment when the finger disk reaches the target stimulus before ShieldActivationDelay during a trial where HasShield == true.

InterTrialInterval

The duration in seconds between the end of a trial and appearance of the disk on the next trial; (because users have to place their finger on the disk, the trials are player-paced).

Type: float Default: 0.5 Range: 0 - Inf

User input parameters (1 parameters)

MaxResponseTime

The duration in seconds during which the player can answer, starting at the moment the stimulus appears on the screen(i.e., start of the trial). In trials where the shield appears around the target and the player must not hit the shield, not hitting the shield during the allotted time should lead to a correct response.

Type: float Default: 2 Range: 0.01 - Inf

Feedback parameters (2 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always

ShowAccuracyOnTargetFeedback

Whether we should show color the stimulus green/red depending on trial success. If false, it will remain white.

Type: boolean Default: "true"
