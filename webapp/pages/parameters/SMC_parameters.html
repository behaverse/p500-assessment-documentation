Parameters

Stimulus parameters (10 parameters)

MatrixSize

The size of the square matrix containing the items.

Type: integer Default: 5 Range: 2 - 10
Note: If ItemCount + 1 > MatrixSize², then MatrixSize will be increased automatically so it can contain all elements (+1 for MoveItem discrepancy type).

ItemCount

The number of items in the pattern.

Type: integer Default: 3 Range: 1 - 25

SymbolPoolCount

The number of different symbols that can be used to generate items.

Type: integer Default: 5 Range: 1 - 5

ColorPoolCount

The number of different colors that can be used to generate items.

Type: integer Default: 1 Range: 1 - 5

InitialItemRotation

Define the set of possible initial rotation for each individual item (i.e., NOT the discrepancy, NOT the Set Transformation).

Type: enum
Note: If InitialItemRotation = None, DiscrepancyType cannot be ChangeSymbol and SymbolPoolCount cannot be more than 1.

HasDiscrepancy

Whether the two sets are equal, or if there is a discrepancy between the two.

Type: boolean Default: Bernoulli(p=0.5)

DiscrepancyType

The type of discrepancy between the two sets.

Type: enum Default: ChangeSymbol; RotateItem for MR

SetTransformation

The transformation applied to the right set of icons.

Type: enum Default: None for PC; RotateClockwise for MR

ShowTransformationHelper

Whether we add indicators (e.g. a black&white bar below the set) on both sets to show the player which transformation has been applied to the second set. (Only for MR).

Type: boolean Default: "false"

AnimateTransformation

Whether we show the transformation using an animation from non-transformed to transformed state (e.g., for rotation 90°: the right panel rotates over 1 second from its base state to its final rotated 90° state) (Only for MR). If false, the transformation is instantaneous.

Type: boolean Default: ShowTransformationHelper
Note: This parameter should likely have the same value as ShowTransformationHelper (if helper are shown, we animate, else we dont).

User input parameters (1 parameters)

MaxResponseTime

The maximum time the user can take to answer.

Type: float Default: 30 Range: 1 - Inf

Feedback parameters (3 parameters)

ShowAccuracyCheckFeedback

Whether we should show the valid or wrong “check” feedback at the end of a trial.

Type: enum Default: Always

ShowDiscrepancyFeedback

Whether we should show the discrepancy if the user missed it at the end of a trial.

Type: enum Default: OnError

ShowAccuracyOnButtonFeedback

Whether the button highlight shows if the user response is correct/incorrect or instead does not reveal response accuracy. If set to True, the highlight is green for correct responses and red for incorrect responses.

Type: boolean Default: "false"
