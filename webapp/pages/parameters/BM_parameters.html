ProblemType

The type algorithm used to create the item/problem.

User input parameters (1 parameters)

MaxIdleTime

The maximum allowed time (in seconds) for the user to enter a response. If he/she does not respond in time, the trial is considered as failed.

Type: float Default: 60 Range: 1 - Inf

Feedback parameters (4 parameters)

ShowAccuracyCheckFeedback

Whether we show the valid or wrong “check” feedback at the end of a trial on the screen center.

Type: enum Default: Always

ShowAccuracyOnButtonFeedback

Whether we color the outline of the ResponseElements red or green to indicate whether the response was correct or incorrect. If false, the outline will be white.

Type: boolean Default: "true"

ShowCorrectTrialResponseFeedback

highlight the ResponseElement that would have been the correct response.

Type: enum Default: Never
Note: Currently the highlighting animation involves a period change of the button size (~20%). We will want to change this in the future and use yellow outlining to be more consistent with other tasks. As this animation is currently not being used; this change in animation is postponed.

ShowPatternCompletionFeedback

Move the ResponseElement that would have been the correct response into the gap of the stimulus matrix.

Type: enum Default: Never
