# Correct Website Structure Based on Project Specification

## Overview
The current webapp implementation is incorrect and contains excessive content not requested in the project specification. This document outlines the **actual** structure required based on `specs/project_spec.md`.

## Key Terminology (from spec)
- **Engine**: A piece of software that can be parameterized to create specific cognitive assessments. One engine typically supports multiple different cognitive assessments.
- **Timeline**: A specific configuration of an engine that sets parameter values and defines user instructions.



## Required Website Structure

### 1. Homepage (`index.qmd`)
**Purpose**: Project overview and navigation entry point
**Required Content**:
- Brief description of Behaverse assessment documentation
**Current Issue**: Contains excessive content with fake engines and timelines



### 2. Engine Directory (`engines.qmd`)
**Purpose**: Comprehensive listing of all assessment engines
**Required Layout**: Grid view with engine cards showing:
- Engine thumbnail (GIF or static image)
- Engine name and category  
- Brief description (50-100 characters) accessible when clicking on the image


### 3. Individual Engine Pages (`engine-<engine_id>.qmd`) accessible via `engines.qmd`
**Required Sections**:

#### Header Section:
- Engine name and category
- Hero image/animation demonstrating the engine
- Brief description (2-3 sentences)
- Demo link (opens in new tab/embedded iframe)

#### Parameters Section: (maybe neested wtithin engine page; e.g., engine-1/parameters.qmd)
- Data source: Parsed from `content/Task spec.xlsx`
- Display format: Searchable/filterable table with columns: Parameter Name, Type, Default Value, Range, Description

#### Timelines Section: (maybe nested within engine page; e.g., engine-1/timelines.qmd)
- list all the timelines available for this engine:
  - Timeline name
  - screenshot
  - short description
  - link to demo
  - link to more details: when clicked opens corresponding timeline detail page

#### Data Dictionary Section: (maybe nested within engine page; e.g., engine-1/data-dictionnary.qmd)
- All variables generated by the engine
- Interactive table with: Variable name, type, description, example values
- Sample dataset download


### 4. Timeline Detail Pages (e.g., engine-1/timelines/timeline_name.qmd)
**Required Content**:
- Title
- Image
- Description
- Detailed scientific rationale
- Complete parameter specifications (using data that will be provided in a config.json file)
- User-facing instructions formatted for clarity
- Embedded or linked demonstration




## Required functioanlity

### Search Functionality
**Requirements**:
- Global search across all engines, parameters, and timelines
- search witin engines, within parameters and within timelines
- Cross-references and linked navigation
- Search response time < 500ms



### Styling
**Requirements**
- Don't use emoticons ever
- use minimalitic, elegant style (minimal use of color)
- use of interesting fonts (e.g., Futura)
